# В чём суть задания?
В начале задания рассказывают, ==что такое маска сети==: как выглядит и для чего нужна. После чего, требуется выполнить задание, если ==известны== некоторые ==данные==(*например, IP - адрес узла и маска сети, или адрес сети и IP - адрес узла*). Чаще всего встречается задание рода: дан IP - адрес узла *...* и маска сети *...*, найдите **количество IP - адресов в сети**, которые имеют в себе количество единиц кратное трём.
# Что нужно понимать для задания?
Важно ориентироваться, что такое ==IP - адрес узла(компьютера), адрес сети, маска сети и побитовая конъюнкция==.
### IP - адрес узла
Это ==уникальный идентификатор==, присваиваемый каждому **устройству** в сети. Устройство может быть любое: компьютер, телефон, умный чайник, многофункциональный промышленный манипулятор и так далее. В этом задании, IP - адреса рассматриваются **исключительно** в ==локальной сети== и в версии IPv4. 
#### Строение IP - адреса версии IPv4:
IP - адрес представляет собой ==**4 байта**==, разделённые между собой точками.
Так как ==байт равен восьми бит==, то каждый байт можно представить как 8 цифр(*0 или 1*), следовательно максимальное значение одного байта ==может достигать **255**==(*так как 2 в 8 степени равно 256 и счёт начинается с нуля*)
Пример: 
	*Десятеричный вид*: 192.168.1.10
	*Двоичный вид*: 11000000.10101000.00000001.00001010
IP - адрес состоит из **двух частей**:
- *Номер сети*: определяет, к какой сети принадлежит устройство.
- *Номер узла(хоста)*: уникальный идентификатор конкретного устройства в этой же сети.
### IP - адрес сети
Это уникальный идентификатор, который ==обозначает **конкретную сеть**==. С помощью IP - адреса сети определяется IP - адрес узла.  Именно ==номер сети== в IP - адресе узла и ==является== IP - адресом сети.
Пример:
	*Десятеричный вид*: 192.168.0.0
	*Двоичный вид*: 11000000.10101000.00000000.00000000
IP - адрес сети тесно связан с маской сети.
### Маска сети
Это **32 - битное** число, которое ==**определяет**==, какая часть IP - адреса **относится** к IP - адресу сети, а какая - к конкретному узлу(устройству). Проще говоря, маска сети работает как шаблон: ==биты, равные единицы в маске, относятся к сети, а биты с нулями - к узлу==. 
**В маске ВСЕГДА сначала идут единицы, после чего нули**. Ситуации наоборот и чередования быть не может.
Пример:
	*Десятеричный вид*: 255.255.254.0
	*Двоичный вид*: 11111111.11111111.11111110.00000000
### Как связаны эти 3 термина
Если выполнить ==побитовую конъюнкцию==(*поразрядное логическое И*) между IP - адресом узла и маской сети, то можно ==получить IP - адрес сети==, который может пригодиться **для выполнения задания**.
Пример побитовой конъюнкции:
	*IP - адрес узла*: 192.168.1.10
(11000000.10101000.00000001.00001010)
	*Маска сети*: 255.255.255.0
(11111111.11111111.11111111.00000000)
	*IP - адрес сети*: 192.168.1.0 (*над соответствующими битами выполняем конъюнкцию*)
(11000000.10101000.00000001.00000000)

Важно знать про то, что ==системой **уже зарезервировано** два IP - адреса==: адрес сети и широковещательный адрес.
Адрес сети был описан выше, ==он имеет **в конце нули**==. Широковещательный адрес же в свою очередь ==имеет после адреса сети **сплошные единицы**==. Эти адреса **НЕ МОГУТ** быть использованы для узлов.
Если задание требует найти ==количество **IP - АДРЕСОВ**==, то широковещательный адрес и адрес сети следует учитывать, а если задание требует найти ==количество **КОМПЬЮТЕРОВ**==, которые можно подключить к сети, эти адреса учитывать **нельзя**.
# Как выполнять задание?
Некоторые типы задач проще сделать на бумаге, когда некоторые лучше делать через код на Python.
## На бумаге:
##### Первый шаг:
==Преобразовать== известные ==данные== в двоичное представление.
##### Второй шаг:
- Если нужно найти ==IP - адрес сети или IP - адрес узла==: использовать **побитовую конъюнкцию**(*или её определение*), чтобы найти недостающую информацию
- Если нужно найти ==маску сети==: как правило, дано два IP - адреса **узлов** в одной сети, следовательно ==сопоставив их==, можно определить маску с наибольшими/наименьшими количествами единиц в двоичной записи.
##### Третий шаг:
==Выполнить требуемое задание==, используя **определение бита**(*то что он может хранить в себе 2 значения*), для поиска количества IP - адресов в сети. Если нужно найти маску с наименьшими/наибольшими количествами единиц, смотреть второй шаг.
## Через код:
##### Первый шаг:
Для работы с IP - адресами, ==подключить библиотеку **ipaddress**==.
##### Второй шаг:
Чтобы получить сеть, имея IP - адрес узла и маску, ==нужно присвоить== **переменной** ==значение== функции **`ip_network("IP-адрес/маска сети",0)`**.
##### Третий шаг:
Чтобы ==перебрать **каждый** IP - адрес== в сети, нужно использовать цикл **`for [переменная цикла] in [переменная сети]`**. Переменная цикла будет принимать значение каждого IP - адреса в сети(***включая** адрес сети и широковещательный адрес*).
##### Четвёртый шаг:
Обычно в заданиях требуется ==работать с двоичным представлением== IP - адресов. Чтобы перевести IP - адрес в двоичный вид, нужно присвоить новой переменной значение функции **`bin(int([переменная цикла]))[:2]`**. **`bin`** переводит IP-адрес, представленный как число при помощи **`int`**(*изначально они имеют свой класс **`IPv4Address`***), в двоичное представление. Полученное ==значение будет являться **строкой**==, следовательно будет не сложно посчитать количество единиц в ней, или условие кратности суммы этих единиц на какое-либо число.
##### Пятый шаг:
Выполнить задание, ==используя полученные данные==, и записать ответ.

### Шаблон на Python
Допустим, сеть задана IP-адресом 122.159.136.144 и маской сети 255.255.255.248. Вопрос, на который нужно дать ответ: сколько в этой сети IP-адресов, для которых количество единиц в двоичной записи IP-адреса не кратно 4?

```
from ipaddress import *

net = ip_network("122.159.136.144/255.255.255.248",0) #создаём сеть, из которой потом будем тащить IP-адреса

ip_count = 0 #счётчик подходящих нам IP-адресов.
for i in net: #перебираем каждый IP-адрес в сети
	ip = bin(int(i))[:2] #переводим IP-адрес в двоичный вид
	if ip.count("1") % 4 != 0: #проверяем условие
		ip_count += 1 #добавляем единицу к счётчику
print(ip_count) #выводим счётчик на экран
```
